Understanding RDMA Request Tail Latency

Project Description

Modern datacenter applications increasingly rely on Remote Direct Memory Access (RDMA) to achieve ultra-low latency and high throughput by bypassing the operating system kernel and enabling direct data placement into application memory. While average RDMA latency can be as low as a few microseconds, tail latency—the high-percentile (e.g., 99th or 99.9th) request completion time—remains a critical challenge. Unpredictable tail latencies can severely impact the performance of large-scale distributed systems, causing request timeouts, head‑of‑line blocking, and degraded user experience.

This project aims to systematically characterize the sources of tail latency in RDMA operations across different transport types (e.g., InfiniBand, RoCE) and network configurations. Through controlled microbenchmarks and real‑world workload traces, we will isolate contributing factors such as congestion, NIC queuing, host‑side processing delays, and firmware interactions. The study will develop analytical models that capture the tail behavior under various load conditions and explore software‑ and hardware‑level mitigation strategies—including adaptive flow control, request scheduling, and NIC offloads—to bound tail latency while preserving high throughput.

The expected outcomes include a comprehensive taxonomy of RDMA tail‑latency bottlenecks, open‑source measurement tools, and practical design guidelines for building predictable RDMA‑accelerated systems. By deepening the understanding of tail latency, this work will enable more reliable and performant datacenter infrastructure for latency‑sensitive applications such as key‑value stores, machine learning training, and distributed transactions.
